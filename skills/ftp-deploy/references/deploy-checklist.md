# FTP Deploy Checklist

只有在需要部署前檢查、排錯或解釋腳本行為時再讀這份檔案。

## 部署前檢查

- 目前工作目錄是否就是要上傳的網站根目錄
- `.env` 是否存在
- `FTP_HOST`、`FTP_USER`、`FTP_PASSWORD` 是否都有值
- `FTP_REMOTE_DIR` 是否通常為 `/httpdocs`
- 遠端主機是否真的是 Plesk 提供的 FTP 主機

## 常見失敗原因

- `.env` 不存在或欄位缺漏
- 帳號密碼錯誤
- 遠端目錄不是 `/httpdocs`
- 主機要求 FTPS，但設定與實際環境不一致
- 本機在錯誤資料夾執行腳本，導致上傳了不該上傳的內容

## 腳本行為摘要

- 讀取目前工作目錄下的 `.env`
- 遞迴上傳目前工作目錄內容
- 會忽略部分目錄與敏感檔案
- 預設目標是 `/httpdocs`

## 建議互動方式

如果使用者需求不完整，先補足以下資訊再執行：
- FTP 主機
- FTP 帳號
- FTP 密碼
- 遠端目錄
- 是否需要安全連線
